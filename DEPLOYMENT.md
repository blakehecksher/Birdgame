# GitHub Pages Deployment (No `gh` CLI Required)

This project is configured for GitHub Pages hosting under:

- Repo name: `Birdgame`
- Vite base path: `/Birdgame/`
- Deploy command: `npm run deploy`

## 1. Create a GitHub repository

1. Open GitHub and create a new repository named `Birdgame`.
2. Do not add a README, `.gitignore`, or license from GitHub UI.

## 2. Connect local repo and push

Run in this project root:

```bash
git remote add origin https://github.com/<YOUR_USERNAME>/Birdgame.git
git add .
git commit -m "Initial commit: Hawk & Pigeon multiplayer game"
git push -u origin main
```

## 3. Publish to `gh-pages`

```bash
npm run deploy
```

This will build and push `dist/` to the `gh-pages` branch.

## 4. Enable Pages

In GitHub repository settings:

1. Go to `Settings -> Pages`
2. Source: `Deploy from a branch`
3. Branch: `gh-pages`, folder `/ (root)`
4. Save

## 5. Open your game

Your live URL will be:

```text
https://<YOUR_USERNAME>.github.io/Birdgame/
```

## Notes

- Room links generated by the host UI are compatible with this URL and include `?room=XXXXXX`.
- If you rename the repository, update `base` in `vite.config.ts`.
